Delhivery Maps API Documentation

Welcome to the Delhivery Maps developer documentation. These APIs provide powerful tools to integrate mapping, routing, and location intelligence into your applications — built specifically for Indian roads, addresses, and scale.

With Delhivery Maps APIs you can display maps, generate optimized delivery routes, geocode addresses, search for places, and power real-time location tracking across India's 18,800+ pin codes.

Route

Optimized directions, distance matrix, fleet planning, and route optimization across India's road network.

Geocode

Convert addresses to coordinates and back with 98%+ accuracy. Handles Indian address formats natively.

Search

Autocomplete, nearby search, place details, and text search across 18,800+ pin codes.

Map Tiles

2D, 3D, and satellite tiles with a style editor for custom-branded map experiences.

Places

Rich POI data, address validation, geofencing, elevation, and photo APIs for location-aware apps.

SDKs

Native Android, iOS, and Web SDKs for maps, navigation, and places with drop-in UI components.

Quickstart

Get up and running in minutes. Follow these steps to make your first API call.

1

Create an account

Sign up at maps.delhivery.com to get access to the developer dashboard.

2

Get your API key

From the dashboard, create a new project and generate your API key. Keep it secure — never expose it in client-side code.

3

Make your first request

Use your API key to authenticate requests. Here's a simple geocoding example:

# Geocode an address
curl -X GET \
  "https://api.delhiverymaps.com/v1/geocode?address=Koramangala+Bengaluru" \
  -H "Authorization: Bearer YOUR_API_KEY"

Authentication

All API requests require authentication using a Bearer token. Include your API key in the Authorization header of every request.

Authorization: Bearer YOUR_API_KEY

For OAuth 2.0 flows, obtain a client key and secret from the dashboard and exchange them for an access token.

Multilingual Language Support

Delhivery Maps APIs support responses in 30+ languages including all major Indian regional languages. Pass the language parameter to receive results in your preferred language.

GET /v1/geocode?address=Koramangala&language=hi  # Hindi
GET /v1/geocode?address=Koramangala&language=kn  # Kannada
GET /v1/geocode?address=Koramangala&language=ta  # Tamil

Supported languages include: English (en), Hindi (hi), Kannada (kn), Tamil (ta), Telugu (te), Bengali (bn), Marathi (mr), Gujarati (gu), Malayalam (ml), Punjabi (pa), and more.

Directions API REST

Calculate optimized routes with turn-by-turn navigation. Built on real delivery data for accurate ETAs across India's road network.

GET
/v1/directions
Get turn-by-turn directions between origin and destination with waypoints
{
  "origin": "12.9716,77.5946",
  "destination": "12.9352,77.6245",
  "waypoints": ["12.9500,77.6100"],
  "mode": "driving",
  "alternatives": true,
  "avoid": ["tolls", "highways"]
}

Directions Basic API REST

A lightweight version of the Directions API for simple A-to-B routing without waypoints. Ideal for high-volume, low-latency use cases.

GET
/v1/directions/basic
Simple point-to-point routing with minimal response payload

Distance Matrix API REST

Calculate travel times and distances between multiple origins and destinations simultaneously. Essential for logistics optimization and delivery planning.

POST
/v1/distance-matrix
Compute an N×M matrix of travel times and distances
{
  "origins": ["12.9716,77.5946", "13.0827,80.2707"],
  "destinations": ["12.9352,77.6245", "12.8456,77.6603"],
  "mode": "driving"
}

Distance Matrix Basic API REST

A simplified distance matrix for 1-to-many or many-to-1 routing scenarios with reduced response size for faster processing.

GET
/v1/distance-matrix/basic
Lightweight distance matrix for simple hub-and-spoke routing

Route Optimizer API REST

Solve the Vehicle Routing Problem (VRP) to find the most efficient sequence of stops for one or more vehicles. Reduces total distance and delivery time.

POST
/v1/route-optimizer
Optimize multi-stop routes for single or multiple vehicles

Fleet Planner API REST

Advanced fleet management with capacity constraints, time windows, driver shifts, and vehicle types. Built for enterprise logistics operations.

POST
/v1/fleet-planner
Plan optimal routes across an entire fleet with constraints

Roads APIs REST

Snap GPS coordinates to the nearest road, get speed limits, and identify road segments. Useful for fleet tracking and telematics applications.

GET
/v1/roads/snap-to-roads
Snap a path of GPS coordinates to the road network
GET
/v1/roads/speed-limits
Get speed limit information for road segments

Geocoding REST

Convert addresses to coordinates and back with 98%+ accuracy. Handles Indian address formats, landmarks, and pin codes natively.

GET
/v1/geocode
Forward geocoding — address to coordinates
GET
/v1/reverse-geocode
Reverse geocoding — coordinates to address

Example response

{
  "status": "OK",
  "results": [{
    "formatted_address": "Koramangala, Bengaluru, Karnataka 560034",
    "geometry": {
      "location": {
        "lat": 12.9352,
        "lng": 77.6245
      }
    },
    "pin_code": "560034",
    "confidence": 0.98
  }]
}

Autosuggest API REST

Real-time, context-aware location suggestions that predict user intent with every keystroke. Optimized for Indian place names, landmarks, and addresses.

GET
/v1/autosuggest
Get location suggestions as the user types
# Example: suggest locations for "Koramang"
GET /v1/autosuggest?query=Koramang&limit=5&location=12.9716,77.5946

Search API REST

Semantic intent analysis that distinguishes addresses from place categories, delivering precise location results instantly.

GET
/v1/search
Search for places, addresses, or categories

Custom-styled, cross-platform map tiles that transform raw coordinates into interactive navigation experiences. Available in raster and vector formats.

GET
/v1/tiles/{z}/{x}/{y}
Fetch map tile at zoom level z, tile x, tile y

Map SDKs

Native SDKs for embedding interactive maps in Android and iOS applications with full support for custom styles, markers, and overlays.

  • Android SDK — Available via Maven Central: com.delhivery:maps-android:1.0.0
  • iOS SDK — Available via Swift Package Manager

Places SDKs

Client-side SDKs for integrating autosuggest, search, and place details directly into your mobile apps with built-in UI components.

  • Android Places SDK — Drop-in search bar with autosuggest
  • iOS Places SDK — Native SwiftUI and UIKit components

Web SDK

A JavaScript SDK for embedding interactive maps in web applications. Compatible with React, Vue, Angular, and vanilla JS.

// Install
npm install @delhivery/maps-js

// Initialize
import { DelhiveryMaps } from '@delhivery/maps-js';

const map = new DelhiveryMaps.Map('map-container', {
  center: [12.9716, 77.5946],
  zoom: 13,
  apiKey: 'YOUR_API_KEY'
});

Delhivery Maps MCP

The Delhivery Maps Model Context Protocol (MCP) server lets AI assistants query mapping data, geocode addresses, and plan routes directly from your AI workflow — no manual API calls needed.

# Add to your MCP config
{
  "mcpServers": {
    "delhivery-maps": {
      "command": "uvx",
      "args": ["delhivery-maps-mcp@latest"],
      "env": { "DELHIVERY_API_KEY": "YOUR_API_KEY" }
    }
  }
}

Error Codes

All errors follow a consistent format with an HTTP status code and a JSON body.

  • 400 — Bad Request: missing or invalid parameters
  • 401 — Unauthorized: invalid or missing API key
  • 403 — Forbidden: quota exceeded or plan restriction
  • 404 — Not Found: resource does not exist
  • 429 — Too Many Requests: rate limit exceeded
  • 500 — Internal Server Error: contact support

Rate Limits

To ensure fair usage and system stability, rate limits are enforced per API key. If you exceed the limit, you'll receive a 429 response.

  • Free tier: 1,000 requests/day, 10 requests/second
  • Professional: 100,000 requests/day, 100 requests/second
  • Enterprise: Custom limits — contact sales@delhiverymaps.com
